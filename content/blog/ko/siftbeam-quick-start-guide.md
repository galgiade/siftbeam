## 서비스 이용 전체 흐름

siftbeam은 기업별로 맞춤화된 데이터 처리를 제공하는 B2B 서비스입니다. 이용 시작까지의 흐름은 다음과 같습니다:

```
1. 계정 생성·회사 정보 등록
   ↓
2. 신규 주문으로 정책(처리 내용) 상담
   ↓
3. 정책 완성(당사가 설정)
   ↓
4. 그룹 생성(사용자와 정책 연결)
   ↓
5. 서비스 페이지에서 파일 업로드·처리
   ↓
6. 처리 결과 다운로드
```

**중요 포인트**:
- siftbeam은 "고객별 맞춤화"가 특징입니다
- 초기에는 처리 내용(정책) 상담·설정이 필요합니다
- 정책 설정 후에는 반복적으로 데이터 처리가 가능합니다

## 사전 준비

시작하기 전에 다음을 준비하세요:

### 필요한 것

- **이메일 주소**: 계정 등록에 사용
- **회사 정보**: 회사명, 주소 등 기본 정보
- **처리할 데이터 파일**: CSV, JSON, Excel 등 (샘플 파일로)

### 권장 환경

- **브라우저**: Chrome, Firefox, Safari, Edge (최신 버전)
- **인터넷 연결**: 안정적인 연결 환경

## 단계 1: 계정 생성 및 회사 정보 등록

### 1-1. 회원가입

[siftbeam 회원가입 페이지](https://siftbeam.com/ko/signup/auth)에 접속합니다.

**입력 항목**:
- **이메일 주소**: 비즈니스용 이메일 주소
- **비밀번호**: 8자 이상, 영숫자 및 기호 포함

이메일 인증 후 로그인 화면으로 이동합니다.

### 1-2. 회사 정보 등록

첫 로그인 시 회사 정보 등록이 필요합니다.

**필수 항목**:
- 회사명
- 국가
- 우편번호
- 시/도
- 시/구
- 주소 1
- 전화번호
- 청구용 이메일 주소

**선택 항목**:
- 주소 2(건물명·호수 등)

### 1-3. 관리자 계정 생성

회사 정보 등록 후 관리자 계정을 생성합니다.

- 이름
- 이메일 주소
- 비밀번호

이것으로 계정 생성이 완료됩니다. 대시보드에 접속할 수 있게 됩니다.

## 단계 2: 신규 주문으로 정책 상담

정책이란 "어떤 데이터 처리를 할 것인가"를 정의한 것입니다. siftbeam은 고객별로 맞춤화된 처리를 제공하므로 먼저 정책 상담이 필요합니다.

### 2-1. 신규 주문 페이지로 이동

1. 대시보드의 "신규 주문" 클릭
2. "생성" 버튼 클릭

### 2-2. 처리 내용 상담

신규 주문 양식에서 다음 정보를 기입합니다:

**필수 항목**:
- **데이터 유형**: 구조화된 데이터, 비구조화된 데이터, 혼합 또는 기타 중 선택
- **모델 유형**: 분류, 회귀, 클러스터링 또는 기타 중 선택
- **제목**: 처리 내용의 간단한 설명
- **상세 내용**: 필요한 데이터 처리에 대한 구체적 기재

**기재 예**:
```
데이터 유형: 구조화된 데이터
모델 유형: 분류
제목: CSV 파일 데이터 변환 처리

상세 내용:
- 입력: 고객 데이터 CSV(이름, 주소, 전화번호 등)
- 처리 내용: 주소 정규화, 전화번호 형식 통일
- 출력: 정형화된 CSV 파일
- 처리 빈도: 주 1회 정도
- 데이터량: 1회당 약 10,000건
```

### 2-3. 샘플 파일 첨부(선택)

처리하고 싶은 데이터의 샘플 파일을 첨부하면 더 정확한 견적이 가능합니다.

- 최대 10개 파일
- 파일당 50MB 이하

### 2-4. 제출 및 확인

양식을 제출하면 당사 담당자가 내용을 확인하고 다음을 수행합니다:

1. **처리 내용 확인**: 요구사항 청취
2. **견적 제시**: 처리 요금 개산
3. **정책 생성**: 처리 흐름 설정
4. **테스트 실행**: 샘플 데이터로 동작 확인

**소요 시간**: 일반적으로 2-5 영업일

### 2-5. 정책 완성 알림

정책이 완성되면 신규 주문 상태가 업데이트됩니다. 대시보드의 "신규 주문" 페이지에서 진행 상황을 확인할 수 있습니다.

정책이 완성되면 다음 단계로 진행할 수 있습니다.

## 단계 3: 그룹 생성

정책이 완성되면 다음으로 그룹을 생성합니다. 그룹은 "어떤 사용자가 어떤 정책을 사용할 수 있는지"를 관리하는 구조입니다.

### 3-1. 그룹 관리 페이지로 이동

1. 대시보드의 "그룹 관리" 클릭
2. "생성" 버튼 클릭

### 3-2. 그룹 정보 입력

**필수 항목**:
- **그룹명**: 알기 쉬운 이름 설정(예: 영업부, 데이터 분석팀)
- **설명**: 그룹의 용도나 목적(선택)

### 3-3. 정책 선택

생성된 정책 중에서 이 그룹에서 사용할 정책을 선택합니다.

- 여러 정책 선택 가능
- 나중에 추가·삭제도 가능

### 3-4. 사용자 추가

그룹에 소속될 사용자를 추가합니다.

**방법 1: 기존 사용자 추가**
- 사용자 관리에서 생성된 사용자 선택

**방법 2: 신규 사용자 생성 후 추가**
- 이름, 이메일 주소 입력
- 초대 메일 발송

### 3-5. 그룹 생성 완료

그룹을 생성하면 소속 사용자는 선택한 정책을 사용하여 데이터 처리를 할 수 있게 됩니다.

## 단계 4: 서비스 페이지에서 파일 처리

그룹 설정이 완료되면 드디어 데이터 처리를 실행할 수 있습니다.

### 4-1. 서비스 페이지 접속

1. 대시보드의 "서비스" 클릭
2. 사용할 정책 선택

### 4-2. 파일 업로드

**방법 1: 드래그 앤 드롭**
- 파일을 업로드 영역에 드래그 앤 드롭

**방법 2: 파일 선택**
- "파일 선택"을 클릭하여 파일 브라우저에서 선택

**제한**:
- 최대 10개 파일
- 파일당 100MB 이하 권장
- 지원 형식: 정책에서 설정된 형식

### 4-3. 처리 시작

1. 파일이 업로드되면 "처리 시작" 버튼 클릭
2. 처리가 자동으로 시작됩니다

### 4-4. 처리 상태 확인

처리 상태는 다음으로 확인할 수 있습니다:

- **상태**: "처리 중", "완료", "오류"
- **처리 시간**: 처리 시작부터의 경과 시간
- **파일 정보**: 파일명, 크기

**처리 시간 기준**:
- 데이터량이나 처리 내용에 따라 다릅니다
- 일반적으로 몇 초~몇 분 정도

### 4-5. 백그라운드 처리

처리 중에도 다음 작업이 가능합니다:

- ✅ 다른 파일 업로드
- ✅ 과거 처리 이력 확인
- ✅ 화면을 닫아도 처리는 계속됩니다

## 단계 5: 처리 결과 확인 및 다운로드

처리가 완료되면 결과를 확인하고 다운로드합니다.

### 5-1. 처리 이력 확인

서비스 페이지 또는 대시보드에서 처리 이력을 확인할 수 있습니다.

**표시되는 정보**:
- **상태**: 처리 완료, 처리 중, 오류
- **처리 일시**: 처리를 실행한 일시
- **파일명**: 처리한 파일명
- **처리 시간**: 처리에 걸린 시간
- **데이터량**: 처리한 데이터의 크기

### 5-2. 결과 다운로드

처리가 완료된 파일은 다음 절차로 다운로드할 수 있습니다:

1. 처리 이력에서 대상 파일 선택
2. "다운로드" 버튼 클릭
3. 파일이 자동으로 다운로드됩니다

**다운로드 가능한 파일**:
- **입력 파일**: 업로드한 원본 파일
- **출력 파일**: 처리 후 파일

### 5-3. 데이터 보존 기간

- **보존 기간**: 업로드 후 1년
- **자동 삭제**: 1년 후 자동으로 삭제됩니다
- **재다운로드**: 보존 기간 중에는 몇 번이든 다운로드 가능

**중요**: 
필요한 데이터는 반드시 로컬에 백업하세요.

### 5-4. 처리 오류의 경우

오류가 발생한 경우 다음을 확인하세요:

1. **오류 메시지 확인**: 원인 파악
2. **파일 형식 확인**: 정책에서 지정된 형식인지
3. **파일 크기 확인**: 제한 내인지
4. **지원팀 문의**: 해결되지 않으면 채팅 지원으로
