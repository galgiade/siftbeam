## Service Workflow Overview

siftbeam is a B2B service that provides customized data processing for each enterprise. The workflow to start using the service is as follows:

```
1. Account Creation & Company Registration
   ↓
2. Request Policy (Processing Configuration) via New Order
   ↓
3. Policy Completion (Configured by Our Team)
   ↓
4. Create Groups (Link Users with Policies)
   ↓
5. Upload Files & Process on Service Page
   ↓
6. Download Processing Results
```

**Key Points**:
- siftbeam's key feature is "customization per customer"
- Initial setup requires policy (processing configuration) consultation
- Once policy is set up, you can repeatedly process data

## Step 1: Account Creation and Company Registration

### 1-1. Sign Up

Visit the [siftbeam Sign-Up Page](https://siftbeam.com/en/signup/auth).

**Required Information**:
- **Email Address**: Business email address
- **Password**: 8+ characters, including letters, numbers, and symbols

After email verification, you'll be redirected to the login screen.

### 1-2. Register Company Information

Company information registration is required on first login.

**Required Fields**:
- Company Name
- Country
- Postal Code
- State/Province
- City
- Address Line 1
- Phone Number
- Billing Email

**Optional Fields**:
- Address Line 2 (Building name, room number, etc.)

### 1-3. Create Administrator Account

After registering company information, create an administrator account.

- Name
- Email Address
- Password

Account creation is now complete. You can access the dashboard.

## Step 2: Request Policy via New Order

A policy defines "what kind of data processing to perform." Since siftbeam provides customized processing for each customer, you first need to request a policy consultation.

### 2-1. Navigate to New Order Page

1. Click "New Order" on the dashboard
2. Click "Create" button

### 2-2. Request Processing Configuration

Fill in the following information in the new order form:

**Required Fields**:
- **Data Type**: Select from Structured Data, Unstructured Data, Mixed, or Other
- **Model Type**: Select from Classification, Regression, Clustering, or Other
- **Subject**: Brief description of processing needs
- **Details**: Specific description of required data processing

**Example**:
```
Data Type: Structured Data
Model Type: Classification
Subject: CSV File Data Transformation

Details:
- Input: Customer data CSV (name, address, phone number, etc.)
- Processing: Address normalization, phone number format standardization
- Output: Formatted CSV file
- Frequency: About once per week
- Data Volume: Approximately 10,000 records per batch
```

### 2-3. Attach Sample Files (Optional)

Attaching sample files of the data you want to process enables more accurate estimates.

- Maximum 10 files
- Up to 50MB per file

### 2-4. Submit and Review

After submitting the form, our team will review the content and:

1. **Confirm Processing Requirements**: Requirement hearing
2. **Provide Estimate**: Approximate processing costs
3. **Create Policy**: Configure processing flow
4. **Test Execution**: Verify operation with sample data

**Time Required**: Typically 2-5 business days

### 2-5. Policy Completion

Once our team creates and completes the policy configuration, the new order status will be updated. You can check progress on the "New Order" page in the dashboard.

Once the policy is complete, you can proceed to the next step.

## Step 3: Create Groups

Once the policy is complete, create groups. Groups are a mechanism to manage "which users can use which policies."

### 3-1. Navigate to Group Management Page

1. Click "Group Management" on the dashboard
2. Click "Create" button

### 3-2. Enter Group Information

**Required Fields**:
- **Group Name**: Set an easy-to-understand name (e.g., Sales Department, Data Analysis Team)
- **Description**: Purpose or use of the group (optional)

### 3-3. Select Policies

Select the policies this group will use from the created policies.

- Multiple policies can be selected
- Can be added or removed later

### 3-4. Add Users

Add users who will belong to the group.

**Method 1: Add Existing Users**
- Select users already created in User Management

**Method 2: Create and Add New Users**
- Enter name and email address
- Invitation email will be sent

### 3-5. Group Creation Complete

Once you create the group, member users can process data using the selected policies.

## Step 4: Process Files on Service Page

Once group setup is complete, you can execute data processing.

### 4-1. Access Service Page

1. Click "Service" on the dashboard
2. Select the policy to use

### 4-2. Upload Files

**Method 1: Drag & Drop**
- Drag and drop files into the upload area

**Method 2: File Selection**
- Click "Select Files" and choose from file browser

**Limits**:
- Maximum 10 files
- Recommended 100MB or less per file
- Supported formats: Formats configured in the policy

### 4-3. Start Processing

1. Once files are uploaded, click "Start Processing" button
2. Processing starts automatically

### 4-4. Monitor Processing Status

You can check processing status:

- **Status**: "Processing", "Complete", "Error"
- **Processing Time**: Elapsed time since processing started
- **File Information**: File name, size

**Estimated Time**:
- Varies depending on data volume and processing content
- Usually takes several seconds to minutes

### 4-5. Background Processing

During processing, you can:

- ✅ Upload other files
- ✅ Check past processing history
- ✅ Close the screen (processing continues)

## Step 5: Review and Download Results

Once processing is complete, review and download the results.

### 5-1. Check Processing History

You can check processing history on the service page or dashboard.

**Displayed Information**:
- **Status**: Processing Complete, Processing, Error
- **Processing Date/Time**: Date and time processing was executed
- **File Name**: Name of processed file
- **Processing Time**: Time taken for processing
- **Data Volume**: Size of processed data

### 5-2. Download Results

You can download completed files with the following steps:

1. Select the target file from processing history
2. Click "Download" button
3. File downloads automatically

**Downloadable Files**:
- **Input File**: Original uploaded file
- **Output File**: File after processing

### 5-3. Data Retention Period

- **Retention Period**: 1 year from upload
- **Auto-Deletion**: Automatically deleted after 1 year
- **Re-Download**: Can be downloaded multiple times during retention period

**Important**: 
Always backup necessary data locally.

### 5-4. In Case of Processing Errors

If an error occurs, check the following:

1. **Check Error Message**: Identify the cause
2. **Verify File Format**: Is it the format specified in the policy?
3. **Check File Size**: Is it within limits?
4. **Contact Support**: If unresolved, contact chat support
